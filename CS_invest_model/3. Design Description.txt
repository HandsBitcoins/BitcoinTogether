프로그램 디자인은...

기초적으로 저의 사상인 모듈화를 철저히합니다. ㅋㅋㅋㅋ

일단 시스템 설명에서도 볼 수 있듯이 수익모델 자체도 네가지로 분리하지만
전체적인 프로그램도 재활용을 염두해두고 다 쪼갭니다.

현재는 모델/API/API와 모델을 쓴 UI로 나눌 생각입니다.
UI는 간단하게 CUI 먼저 하구요, GUI는 차후 할 생각입니다. 사실 할 생각이 없습니다... ㅋㅋㅋㅋ
다른 수익 모델을 돌릴 생각이 없어서 수익 모델을 교체할 수 있게 만들 생각은 없습니다.


API는 자체적으로 완성만 된 DLL처럼 함수에 입력만 받으면 움직이게 끔 만들 생각입니다.
뭐, 더 설명할게 없네요.


모델 파일에서 API를 사용하는 애는 딱 두 가지입니다. 
세 가지이지만 예언자는 아직 구현할 생각이 없으니. 두 개.
하나는. 통계학자. 얘는 매수호가, 매도호가를 알아봐야 하니 API가 필요합니다.
나머지 하나는 외부 거래자, 얘는 실제 매수나 매도를 실행해야하니 API가 필요합니다.

모델에서 API에 접근하는 부분은 별도로 만들어서 
차후 코빗 API말고 엑코같은 곳에서도 돌릴 수 있을 때 쉽게 교체 가능하도록 할 생각입니다.


UI는 API와 모델을 적극 이용하는 쪽으로 갑니다.
아마 거의 붙여서 뜯어낼 수 없을 수준으로 갈 거 같습니다.
뭐 API 클래스 이름만 바꾸면 될 정도로 만들 수는 있겠지만
자신이 없네요.

UI에서 모델 쪽에 접근하는 것도 
모델에서 API에 접근 하는 것 같이 
따로 접근용으로 하나의 클래스로 만들 생각입니다.
